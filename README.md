物流配送管理系統
使用說明書
一、系統簡介

本系統為一套以 C++ 語言 開發之「物流配送管理系統」，主要功能為模擬物流公司在日常營運中對 訂單 與 配送人員 的管理流程。
使用者可透過主控台介面完成訂單建立、配送員建立、訂單指派與訂單查詢等基本物流管理功能。

本系統適合用於：

課程作業

系統設計與實現示範

物流資訊系統基礎模型

二、系統執行環境
1. 硬體需求

一般個人電腦即可

2. 軟體需求

作業系統：Windows 10 / 11

開發工具：Visual Studio 2019 或 2022

專案類型：C++ Console Application

三、系統功能說明

本系統提供以下四項主要功能：

新增訂單

新增配送員

指派訂單

顯示訂單列表

所有操作皆透過鍵盤輸入完成。

四、系統操作說明
（一）啟動系統

在 Visual Studio 中執行程式

系統啟動後，畫面會顯示主選單：

物流配送管理系統
1. 新增訂單
2. 新增配送員
3. 指派訂單
4. 顯示訂單
0. 離開
選擇功能：


使用者請輸入對應的數字來選擇功能。

（二）新增訂單（功能選項 1）

操作步驟：

在主選單輸入 1

依序輸入：

訂單編號

客戶名稱

配送地址

系統回應：

訂單新增成功


說明：

新增後的訂單狀態預設為「待配送」

尚未指派配送員

（三）新增配送員（功能選項 2）

操作步驟：

在主選單輸入 2

輸入：

配送員編號

配送員姓名

系統回應：

配送員新增成功

（四）指派訂單（功能選項 3）

操作步驟：

在主選單輸入 3

輸入：

訂單編號

配送員編號

系統回應：

訂單已指派給 XXX


說明：

訂單狀態會由「待配送」變更為「配送中」

若輸入的訂單或配送員不存在，系統會提示錯誤訊息

（五）顯示訂單列表（功能選項 4）

操作步驟：

在主選單輸入 4

顯示內容包含：

訂單編號

客戶名稱

訂單狀態

配送員姓名（或顯示「未指派」）

顯示範例：

--- 訂單列表 ---
訂單：O1001 | 客戶：陳先生 | 狀態：配送中 | 配送員：王小明
訂單：O1002 | 客戶：林小姐 | 狀態：待配送 | 配送員：未指派

（六）離開系統（功能選項 0）

在主選單輸入 0，系統即結束執行。

五、系統限制說明

本系統資料僅暫存於程式執行期間，程式結束後資料不會保留

未使用資料庫系統，僅作為邏輯與流程示範

未實作帳號登入與權限管理功能

六、未來可擴充方向

本系統可進一步擴充以下功能：

資料庫儲存（MySQL / SQLite）

配送完成狀態管理

配送時間紀錄

圖形化介面（GUI）

Web 版物流管理平台

七、結論

本物流配送管理系統透過 C++ 物件導向方式實作，成功模擬物流作業中常見的訂單與配送管理流程，具備基本的實用性與擴充彈性，適合作為物流資訊系統設計與實現之入門範例。
