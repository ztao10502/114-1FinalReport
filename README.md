# 物流配送管理系統

## 一、需求說明書（Software Requirement Specification, SRS）

### 1.1 文件目的

本需求說明書用以描述「物流配送管理系統」之功能需求、非功能需求及使用者介面需求，作為系統設計與實作之依據，確保系統開發符合使用者需求。

### 1.2 系統背景與目標

隨著電子商務發展，物流配送管理成為企業營運中重要的一環。本系統目的在於建立一套簡易且具代表性的物流配送管理系統，協助管理人員有效掌握訂單與配送人員的指派狀態。

### 1.3 使用者角色

使用者角色	說明

系統管理者	負責新增訂單、配送員及指派訂單

### 1.4 功能性需求

#### FR-1 新增訂單

使用者可輸入訂單編號、客戶名稱及配送地址

系統須將訂單狀態設定為「待配送」

#### FR-2 新增配送員

使用者可新增配送員基本資料（編號、姓名）

#### FR-3 指派訂單

使用者可將指定訂單指派給指定配送員

指派成功後訂單狀態須更新為「配送中」

#### FR-4 查詢訂單

使用者可查看所有訂單資訊與配送狀態

### 1.5 非功能性需求

系統需在一般個人電腦上順利執行

操作流程須簡單明確

系統回應時間應即時（小於 1 秒）

系統具備基本錯誤提示機制

### 1.6 使用者介面需求

採用文字型主控台介面

以數字選單方式進行功能操作

輸出內容需清楚顯示訂單與配送資訊

## 二、概要設計說明書（System Design Specification）

### 2.1 系統架構設計

本系統採用 單層式（Single-tier）架構，整合使用者介面、系統邏輯與資料管理於同一程式中，適合作為小型系統與教學示範。

### 2.2 系統模組劃分

模組名稱｜功能說明

訂單管理模組｜負責訂單資料建立與管理

配送員管理模組｜負責配送員資料建立

配送指派模組｜處理訂單與配送員之關聯

系統控制模組｜負責主選單與流程控制

### 2.3 類別設計概述

#### Order 類別

儲存訂單基本資訊

管理訂單狀態與配送員名稱

#### Courier 類別

儲存配送員基本資料

#### LogisticsSystem 類別

作為系統核心

管理訂單與配送員資料集合

提供操作介面

### 2.4 系統流程概述

1.使用者啟動系統

2.顯示主選單

3.使用者選擇功能

4.系統執行對應模組

5.回到主選單或結束程式

## 三、詳細設計說明書（Detailed Design Specification）

### 3.1 類別與資料結構設計

#### Order 類別

成員名稱	資料型態	說明

orderId	string	訂單編號

customer	string	客戶名稱

address	string	配送地址

status	string	訂單狀態

courierName	string	配送員姓名

#### Courier 類別

成員名稱	資料型態	說明

courierId	string	配送員編號

name	string	配送員姓名

#### LogisticsSystem 類別

使用 vector<Order> 儲存訂單資料

使用 vector<Courier> 儲存配送員資料

### 3.2 主要功能流程設計（以指派訂單為例）

使用者輸入訂單編號與配送員編號

系統搜尋訂單集合

系統搜尋配送員集合

若皆存在，更新訂單資料

回傳指派成功訊息

### 3.3 錯誤處理設計

若找不到訂單或配送員，顯示錯誤訊息

防止非法選單輸入造成系統中斷
